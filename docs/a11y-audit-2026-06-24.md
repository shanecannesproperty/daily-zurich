## Keyboard-only verification

All focus stops render the brand focus ring
(`outline: 2px solid rgb(163, 45, 45)` from `:focus-visible`).

### Header + primary nav (homepage, viewport 1280)

```
Tab 1   → Subscribe (top strip)
Tab 2   → The Daily Canberra home (masthead)
Tab 3   → Home
Tab 4   → Live
Tab 5..13 → News, Federal, Finance, Community, Sport, Business, Events,
            This weekend, Best of
Tab 14  → Directory
(continues into Ask Canberra, Property, Obituaries)
```

Order follows DOM source. Every stop is a real `<a>`, has an accessible
name from text or `aria-label`, and shows a visible focus ring.

### Audio briefing controls

The `<figure>` itself is not focusable (correct per HTML spec). Native
`<audio controls>` exposes its play/pause/seek/volume buttons via the
browser shadow DOM and is reachable in DOM order. Verified the
`aria-label` reads:

```
Play audio briefing: Canberra in 5 minutes, 5m 12s, Wed 24 June 2026
```

Below the audio, the new transcript disclosure (`<summary>` inside
`<details>`) is a keyboard-focusable stop with the accessible name
"Read the briefing transcript for {heading}". Enter/Space toggles the
transcript region, which exposes `role="region"` + label "Audio
briefing transcript" for screen readers.

### Newsletter input

```
Focus #newsletter input[type=email]   → email input (sr-only label "Email address")
Tab 1 → Subscribe button (type=submit)
Tab 2 → Privacy Policy link inside the form footer
```

Input has a visible focus ring inside the field (`outline: 2px solid
--ink-red, outline-offset: -1px` from `.field:focus`).

### Admin gate at `/admin/agents`

`/admin/agents` is `ssr: false` and the component returns `null` until
the Supabase session resolves. Unauthenticated keyboard users therefore
land on an empty `<main>` with only chrome links reachable. The
existing `/admin/login` route is the supported entry point and has a
fully focusable email + password + submit chain (Tabs 1 → 2 → 3 with
the brand focus ring). Pre-existing gate behaviour is untouched.
